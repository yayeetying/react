Intro to React

  • Javascript library
  • build website's user interface
  • open-source, free, and popular!
  • maintained by Facebook; used by Instagram, Discord, Netflix

React is like an HTML rendering engine

write Javascript to describe your page's HTML, and React changes the page's DOM accordingly


pros cons
  • no flipping between HTML and JS files
  • allows for modular code through "components"
  • don't need to manually change DOM
  • learning curve for developer
  • extra abstraction (React renders HTML) --> slower
  • React ~38kb library

Installing React

$ npx create-react-app

  • development environment
  • live reload -- code, BAM, changes updated already
  • server at localhost:3000

Render HTML

To render HTML with React, we need to create a rendering root that tells React this is where the HTML will be. This root can be created with a <div>, <p>, or any other tag, using ReactDom.createRoot()

ReactDom.createRoot()

  • function takes one input: a DOM node, ie. "root"
  • returns JS object

root.render()

  • takes one input: a React Element
  • elements created with React.createElement()
